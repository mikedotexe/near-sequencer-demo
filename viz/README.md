# viz/ ‚Äî recipe-book animation scenes

Short companion to [`DESIGN.md`](./DESIGN.md). Read that for the
design rationale; this doc is for rendering quickly.

## What's in here

One Manim scene per NEP-519 recipe, plus a Live variant that replays
the same scene against real testnet captures:

- `scenes/recipe_basic.py` ‚Üí classes `RecipeBasic` (synthetic) and
  `RecipeBasicLive` (live data).
- `scenes/recipe_timeout.py` ‚Üí `RecipeTimeout` and `RecipeTimeoutLive`.
- `scenes/recipe_chained.py` ‚Üí `RecipeChained` and `RecipeChainedLive`.
- `scenes/recipe_handoff.py` ‚Üí `RecipeHandoffClaim` / `RecipeHandoffClaimLive`
  and `RecipeHandoffTimeout` / `RecipeHandoffTimeoutLive` (one scene
  file; two class pairs for the two outcomes).

All primitives (sphere, satellite, TimelinePlayer, palette, blooms,
teach cards, typography) come from `common/`, vendored from
[`../../manim-visualizations`](../../../manim-visualizations). See
[`common/ATTRIBUTION.md`](./common/ATTRIBUTION.md) for the source and
re-sync path.

## Render

```sh
cd viz/
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# Per-recipe synthetic scenes (~15‚Äì60 s each at -ql):
make recipe-basic
make recipe-timeout
make recipe-chained
make recipe-handoff-claim
make recipe-handoff-timeout

# All of them:
make all-recipes

# High quality (slower, for sharing):
make recipe-basic-hq
```

## Live data variants

Each scene has a `*Live` class that reads a translator-generated
timeline JSON built from a real testnet capture. Regenerate the live
JSONs after a fresh `scripts/demo.sh run <recipe>`:

```sh
./scripts/onchain-to-timeline.mjs \
    --raw ../artifacts/testnet/recipe-basic/run-01.raw.json \
    --out data/recipe-basic-live-01.json
manim -ql scenes/recipe_basic.py RecipeBasicLive
```

Same for `recipe-timeout` and `recipe-chained`.

## Data files

- `data/recipe-{basic,timeout,chained}-synthetic.json` ‚Äî hand-authored
  timelines with pedagogical block spacing. The timeout scene
  compresses the 200-block wait via `idle_block_seconds=0.018` in
  `recipe_timeout.py`.
- `data/recipe-handoff-{claim,timeout}-synthetic.json` ‚Äî one per
  ending of Recipe 4.
- `data/recipe-{basic,timeout,chained}-live-NN.json` ‚Äî generated by
  `scripts/onchain-to-timeline.mjs` from `artifacts/.../run-NN.raw.json`.
- `data/recipe-handoff-{claim,timeout}-live-NN.json` ‚Äî generated from
  `artifacts/.../run-{claim,timeout}-NN.raw.json`.

## Event vocabulary (quick reference)

All three scenes use the same TimelinePlayer event types. See
[`DESIGN.md`](./DESIGN.md) for the full table; the short version:

- `actor_appear` ‚Äî bring a sphere on stage
- `tx_included` ‚Äî caller signs a tx; arrow lands on receiver
- `yield_eject` ‚Äî yielded callback receipt shown as an orbiting
  satellite
- `resume_data` / `resume_action` ‚Äî the payload arrives and the
  callback fires
- `downstream_call` / `downstream_return` ‚Äî chained recipe's call to
  counter + return of observed value
- `settle` with status `ok` / `timeout` ‚Äî satellite resolves (green
  bloom) or withers (red ember)
- `narrative` ‚Äî teach-card overlay

## Layout

Each scene file has a `_LAYOUT` dict mapping actor_id to 3D position.
Keep sphere positions such that body bboxes + orbit radii don't
overlap at `orbit_radius=1.10`. The player asserts no-overlap as fatal
‚Äî a cramped layout will crash the render rather than letting actors
collide silently.
